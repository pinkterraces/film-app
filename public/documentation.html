Documentation

Purpose

This app allows users to create a profile and save a list of films they're interested in, as well as browse and find information on similar films.

The API uses Node.js to interface with a database.

API Endpoints

Get list of all movies
Purpose / Business Logic Retrieve all films in database
Resource URL /movies
Parameters None
HTTP Method GET
Request data format UTF-8 encoded URL
Response data format JSON object with list of all films and their data

/movies

Create New User
Purpose / Business Logic Create a new user account in the database
Resource URL /users
Parameters None
HTTP Method POST
Request data format URL with username and JSON providing mandatory user details (Username, Password, Email)
Response data format JSON with user data + newly created user ID
Examples
Example Request 

  {
    "Username": "John Schmitt",
    "Password": "John123",
    "Email": "john.schmitt@email.com"
    "Birthdate": "12.31.86"
  }
Example Response 
          
  {
    "Username": "John Schmitt",
    "Password": "John123",
    "Email": "john.schmitt@email.com",
    "Birthdate": "1986-12-30T23:00:00.000Z",
    "FavoriteMovies": [],
    "_id": "63b5a7ea77dd4e7018e2deb8",
    "__v": 0
  }

Add to favourites
Purpose / Business Logic Adds movie to users list of favourite movies
Resource URL /users/[Username]/favourites/[movieID]
Parameters Username
MovieID
HTTP Method POST
Request data format UTF-8 encoded URL with user id and title of movie
Response data format Text confirmation with name of list + film title added JSON File
Examples
Example Request /users/John%20Schmitt/movies/639d9725656bde24013e68f3
Example Response "Movie added to your favourites!"
